# Skill Panel v2.0.1 批量管理与锁定设计

日期：2026-06-18

## 目标

1. 强化卡片拖拽过程中的位移、目标位置和交换反馈。
2. 区分系统永久锁定与用户可切换锁定，避免误编辑。
3. 支持全部 Skill、当前类目和手动勾选三种批量选择范围。
4. 确保正文修改与删除操作真实作用于电脑中的 Skill 文件或目录。
5. 删除左侧“需要关注”，简化治理信息。
6. 版本统一升级为 v2.0.1，并更新桌面程序、迁移包和产品复盘文档。

## 交互方案

### 拖拽排序

- 卡片按下并移动超过阈值后进入拖拽态，提升阴影、缩放并降低少量透明度。
- 指针经过同类目目标卡片时，目标卡片显示蓝色插入边框和横向位移。
- 松开后卡片顺序立即交换，卡片区域使用短时位移动画强化结果反馈。
- 拖拽只在同一类目内生效，并继续持久化到 `categorySkillOrder`。

### 锁定规则

- 插件缓存、系统 Skill、受保护 lark Skill 永久锁定，右键菜单只显示锁定状态，不提供解锁。
- `codex-user`、`agents-user`、`custom` Skill 默认解锁，用户可以通过右键菜单或批量工具栏锁定、解锁。
- 用户锁定状态写入设置文件的 `skillLocks` 字段，重启应用和电脑后保持。
- 锁定会禁用正文、名称、描述编辑；删除仍通过确认弹窗独立授权。

### 批量选择与操作

- 工具栏新增“批量选择”按钮。进入后，卡片和列表行显示复选框。
- 批量栏提供“全选当前结果”“选择当前类目”“清空选择”。当前搜索和筛选条件会限定全选范围。
- 批量操作包括：更改类目、添加标签、设置卡片颜色、锁定、解锁、删除。
- “更改”定义为类目归属变更。Skill 名称、描述和 Markdown 正文保持单个编辑，避免用同一内容覆盖多个文件。
- 对包含永久锁定 Skill 的批量解锁操作，只处理可切换锁定的 Skill，并在结果状态中显示跳过数量。
- 批量删除显示所选数量和名称摘要，确认后逐个调用真实删除命令；失败项保留在列表并显示错误。

## 数据与文件边界

- `skillLocks: Record<string, boolean>` 保存用户锁定状态，仅接受可写来源路径。
- 类目、标签、颜色和排序属于应用设置，继续保存在 Tauri 设置文件中。
- 名称、描述、Markdown 使用 `update_skill` 写入磁盘 `SKILL.md`，写入前创建备份。
- 删除使用 `delete_skill` 删除电脑中的 Skill 目录，删除前创建可恢复备份。
- 批量删除前端逐项等待命令结果，完成后重新执行 `scan_skills`，界面以磁盘扫描结果为准。

## 组件与状态

- `AppSettings` 增加 `skillLocks`，TypeScript 和 Rust 模型保持 camelCase 序列化契约。
- `App` 增加 `selectionMode`、`selectedSkillPaths`、`dragOverSkillPath` 和批量弹窗状态。
- 现有右键 Skill 菜单增加锁定/解锁命令，并根据永久锁定状态控制可用性。
- 左侧治理区删除“需要关注”，保留用户可编辑和只读 Skill 入口。

## 错误处理

- 设置保存失败沿用全局设置错误提示。
- 批量删除收集每个路径的错误，成功项从选择集合移除，失败项保持选中。
- 真实文件更新失败时保留编辑内容，详情区显示操作错误。
- 删除完成后扫描结果仍存在的路径视为失败，避免界面提前隐藏磁盘文件。
