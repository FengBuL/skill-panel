# Skill Panel 产品问题复盘

更新时间：2026-06-17

## 复盘范围

本文记录 Skill Panel 在类目、卡片、详情、Markdown、解析、桌面打包与迁移交付中暴露的问题，以及本轮修复方式。目标是为后续产品迭代留下可追溯的 PM 复盘材料。

## 问题与解决方式

## 产品层总结

1. 类目需要作为一等数据模型存在，标签只能做补充标记。否则用户看到的导航、分组、筛选会产生不一致。
2. 面向普通用户的界面不应暴露实现来源，技术来源保留在后台用于权限和安全判断即可。
3. Markdown 是 Skill 的核心内容，详情页应以标准预览和稳定编辑体验为中心，辅助模块要克制。
4. 本地桌面应用的更新需要处理进程锁定，打包前应检查旧进程是否仍在运行。
5. 复杂 Skill 文件要容忍真实世界的 frontmatter 差异，读取失败会直接破坏用户信任。
6. 桌面 WebView 的拖拽事件兼容性要单独验证，关键交互需要 pointer 级兜底。
7. 个性化设置必须按“最新用户意图”保存，异步请求乱序是本地应用也会遇到的真实问题。
8. 语言设置属于全局身份体验，任何局部 UI 偏好保存都不能顺手改写语言。
