## 使用边界

- `current/` 中的内容是新任务基线，修改后需要同步相关状态与验证证据。
- `versions/` 中的内容按历史原貌保留，目录内旧路径代表当时项目结构。
- 发布二进制、截图和机器报告位于 `output/`，入口见 [产物地图](../output/README.md)。
